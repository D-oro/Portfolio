Three things every developer should know

Basic accessibility skills no one taught you

This is a summary of all the things I wish someone told me about when I first got started writing accessible code. It's primarily aimed at software developers and frontend designers who want to get better at web accessibility.

1. Using Firefox Accessibility Inspector

Did you know Firefox has integrated accessibility testing tools? There's no need to install any plugins or anything, Firefox comes with an Accessibility Inspector by default. Here's where to find it:

  • Go to any webpage on Firefox
  • Right click
  • Select "Inspect Accessibility Properties"
    (the second option from below )

Firefox context menu

The accessibility inspector contains a lot of useful tools, including automated testing, a colour perception simulator, visually showing the tabbing order, and the accessibility tree of the whole page which can be printed to JSON. It's a great tool for a quick colour contrast check, and to inspect element properties in the accessibility tree.

Firefox Accessibility Inspector

Check out → the Firefox Accessibility Inspector documentation to learn more about this usefull tool.

2. Using keyboard navigation

A great website doesn't rely on a mouse for navigation. Ensuring good keyboard navigation can make interacting with your webpage faster and easier for a lot of people. However a lot of developers don't know the keyboard controls for native html elements, so let's change that! Here are the most common keyboard controls so you can

3. Using a screen reader

The following key combindations can be used to turn the screen reader on and off:

  • Windows Narrator: Ctrl + Windows key + Enter
  • VoiceOver on macOS: command + F5
